# Intermediate Representation

Dexter passes data between agents and tools through typed **Intermediate Representations (IRs)** — JSON artifacts with defined schemas. Each IR is produced by one agent or script and consumed by the next step.

## IR overview

| IR name | File | Producer | Consumer |
|---------|------|----------|----------|
| **Parts IR** | `parts.json` | analyze agent | generate_components, initialize_placement |
| **Layout IR** | `assembly.json` | initialize_placement.py / apply_critic.py | blender_assemble, compute_world_dims, USD export |
| **Critique IR** | `critic.json` | critic agent | apply_critic.py, orchestrator |
| **Placement init** | `placement_init.json` | initialize_placement.py | iteration 001 assembly |
| **World dims** | `world_dims.json` | compute_world_dims.py | critic |
| **Component dims** | `component_dims.json` | generate_components.py | initialize_placement, compute_world_dims |
| **Render views** | `render_views.json` | orchestrator | blender_render_views.py |

## Parts IR (`parts.json`)

Structured output of the **analyze** subagent: the part list and kinematic tree.

```json
{
  "object": "dishwasher",
  "parts": [
    {
      "name": "cabinet_body",
      "description": "The main dishwasher cabinet with insulated walls.",
      "parent": null,
      "joint_type": "fixed"
    },
    {
      "name": "front_door",
      "description": "The front drop-down door panel.",
      "parent": "cabinet_body",
      "joint_type": "revolute",
      "size_fraction": [1.00, 0.06, 0.60],
      "position_in_parent": "center",
      "hinge_side": "bottom"
    }
  ]
}
```

**Key fields for downstream steps:**

- `name` → file stem for PNGs, GLBs, and assembly links
- `description` → copied into image generation prompts
- `size_fraction` + `position_in_parent` → placement hint computation
- `hinge_side` / `slide_axis` → open pose calculation

Approved by human gate before 3D generation begins.

## Layout IR (`assembly.json`)

Layout IR: position, orientation, and scale for each link. Iteration 001 comes from `initialize_placement.py`; later iterations from `apply_critic.py`.

```json
{
  "root": ".intermediate/dishwasher/001",
  "robot_name": "dishwasher",
  "links": [
    {
      "name": "cabinet_body",
      "parent": null,
      "visual_mesh": "component_glbs/cabinet_body.glb",
      "collision_mesh": "component_glbs/cabinet_body.glb",
      "origin": {"xyz": [0.0, 0.0, 0.425], "rpy_deg": [0.0, 0.0, 0.0]},
      "scale": [1.7, 1.6, 1.8]
    }
  ]
}
```

This single IR drives:

1. **Blender assembly** — visual scene for rendering
2. **Critic comparison** — world-space dimensions from scale + raw dims
3. **USD export** — geometry for Isaac Sim

### Transform chain

`blender_assemble.py` applies:

```
world_pos = parent_scale × (origin_xyz + child_scale × raw_mesh_center)
```

**Scale inheritance (non-negotiable):**

```
child_scale[axis] = target_world_size[axis] / (parent_scale[axis] × raw_GLB_size[axis])
```

**Back-computing origin from target world position:**

```
origin_xyz[axis] = target_world_center[axis] / parent_scale[axis]
                   − child_scale[axis] × raw_mesh_center[axis]
```

With rotation (`rpy_deg = [0, 0, θ]`):

```
rotated = R_Z(θ) × (child_scale × raw_mesh_center)
origin_xyz = target_world_center / parent_scale − rotated
```

### Door rotation conventions

| Hinge type | Axis | Sign |
|-----------|------|------|
| Left-hinged door | `rpy_deg[2]` (Z) | Negative opens forward |
| Right-hinged door | `rpy_deg[2]` (Z) | Positive opens forward |
| Bottom-hinged door | `rpy_deg[0]` (X) | Positive drops down |
| Top-hinged lid | `rpy_deg[1]` (Y) | Positive opens backward |

## Critique IR (`critic.json`)

Structured output of the **critic** subagent: score and per-component layout fixes.

```json
{
  "iteration": 3,
  "score": 78,
  "pass": false,
  "summary": "Door angle too shallow; rack position good.",
  "issues": [
    {
      "component": "front_door",
      "problem": "Open angle ~30° but source shows ~45°.",
      "suggested_rotation_delta": [15.0, 0.0, 0.0]
    },
    {
      "component": "cabinet_body",
      "locked": true,
      "problem": "Correct size, position, no collisions."
    }
  ],
  "per_view_notes": {
    "front": "Door angle too shallow.",
    "top": "Alignment OK.",
    "left": "Rack depth matches.",
    "isometric": "Overall shape good."
  }
}
```

Corrections are in **parent-local space**:

```
origin_delta[axis] = Δworld[axis] / parent_scale[axis]
```

## Placement init (`placement_init.json`)

Pre-computed scales and poses from `initialize_placement.py`. Used to build `iterations/001/assembly.json`. All values are derived from `parts.json` geometry fields. If init looks wrong, fix `parts.json` and delete `placement_init.json` plus `iterations/001/assembly.json` to regenerate.

## Component dims (`component_dims.json`)

Raw GLB bounding boxes before any scale:

```json
{
  "parts": {
    "front_door": {
      "size": [0.28, 0.05, 0.33],
      "center": [0.0, 0.0, 0.17],
      "min": [-0.14, -0.025, 0.005],
      "max": [0.14, 0.025, 0.335]
    }
  }
}
```

Never compare raw dims to the source image directly — always compute world size:

```
world_size[axis] = parent_scale[axis] × child_scale[axis] × raw_size[axis]
```

## Data flow chain

```
source.png
  → parts.json              (analyze)
  → component_images/       (generate_components.py)
  → component_glbs/         (generate_components.py)
  → component_dims.json     (generate_components.py)
  → placement_init.json     (initialize_placement.py)
  → assembly.json           (initialize_placement.py / apply_critic.py)
  → world_dims.json         (compute_world_dims.py)
  → critic.json             (critic)  → apply_critic.py (next iteration)
  → assembled.blend         (blender_assemble.py)
  → robot.usda              (blender_export_usd.py)
```
