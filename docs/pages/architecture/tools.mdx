# Tool Scripts

Deterministic Python and Blender scripts handle everything that does not need LLM judgment: image APIs, mesh generation, measurement, scene assembly, rendering, critic correction, and USD export.

The orchestrator runs them via bash. API keys come from environment variables — never from config files.

| Script | Input → Output |
|--------|----------------|
| `generate_components.py` | `parts.json` + `source.png` → `component_images/`, `component_glbs/`, `component_dims.json` |
| `initialize_placement.py` | `parts.json` + `component_dims.json` → `placement_init.json`, `iterations/001/assembly.json` |
| `update_placement.py` | `assembly.json` + `critic.json` → next `assembly.json` |
| `blender_assemble.py` | `assembly.json` (+ `component_dims.json`) → `assembled.blend` |
| `blender_render_views.py` | `assembled.blend` + `render_views.json` → `renders/*.png` |
| `blender_export_usd.py` | `assembled.blend` → `robot.usda`, `robot_prim_map.json`, `textures/` |
| `blender_measure_glbs.py` | `component_glbs/` → `component_dims.json` (called by `generate_components.py`) |
| `common.py` | shared helpers; `--schema` + `--data` → pass or fail |

---

## generate_components.py

| | |
|---|---|
| **Input** | `parts.json`, `source.png`, `configs/base.yaml` |
| **Output** | `component_images/<part>.png`, `component_glbs/<part>.glb`, `component_dims.json` |
| **Requires** | `OPENAI_API_KEY`, `FAL_KEY`, Blender |

**Logic:** For each part, builds an image prompt from the description, calls OpenAI image edit with `source.png` as reference, sends the PNG to fal.ai for a GLB, then measures raw bounding boxes. Skips existing outputs; skips the whole step if `component_dims.json` exists.

```bash
python3 tool_scripts/generate_components.py --run-dir .intermediate/dishwasher/001
```

---

## initialize_placement.py

| | |
|---|---|
| **Input** | `parts.json`, `component_dims.json`, `configs/base.yaml` |
| **Output** | `placement_init.json`, `iterations/001/assembly.json` |

**Logic:** Combines world-space size and pose from `parts.json` with raw mesh sizes from `component_dims.json` to compute each part's Blender `node_scale` and `node_origin`. Both outputs are skipped if they already exist. Validates both output files against their schemas.

```bash
python3 tool_scripts/initialize_placement.py --run-dir .intermediate/dishwasher/001
```

---

## update_placement.py

| | |
|---|---|
| **Input** | Previous `assembly.json`, previous `critic.json` |
| **Output** | Next `assembly.json` (validated against `assembly.schema.json`) |

**Logic:** Copies the previous layout and applies critic corrections directly:

- `corrected_world_size` → replaces `world_size`
- `corrected_world_center` → replaces `world_center`
- `suggested_rotation_delta` → added to `euler_deg`
- `locked: true` → unchanged
- No entry → unchanged

After a score regression, pass the best-scoring layout as `--prev-assembly` instead of the most recent one.

```bash
python3 tool_scripts/update_placement.py \
  --prev-assembly iterations/001/assembly.json \
  --critic        iterations/001/critic.json \
  --output        iterations/002/assembly.json
```

---

## blender_assemble.py

| | |
|---|---|
| **Input** | `assembly.json` (loads `component_dims.json` via the `root` path inside it) |
| **Output** | `assembled.blend` |

**Logic:** Imports each GLB, parents parts in the kinematic tree, and positions each mesh so its world bounding box matches the part's `world_size`, `world_center`, and `euler_deg`. Raw GLB dimensions are used internally to derive Blender scale and pivot.

```bash
blender --background --python tool_scripts/blender_assemble.py -- \
  --layout iterations/001/assembly.json \
  --output iterations/001/assembled.blend
```

---

## blender_render_views.py

| | |
|---|---|
| **Input** | `assembled.blend`, `render_views.json` |
| **Output** | `renders/*.png` |

**Logic:** Renders one PNG per camera entry (front, top, left, isometric). `render_views.json` is written by the orchestrator, not by a subagent.

```bash
blender --background --python tool_scripts/blender_render_views.py -- \
  --blend    iterations/001/assembled.blend \
  --cameras  iterations/001/render_views.json \
  --output-dir iterations/001/renders/
```

---

## blender_export_usd.py

| | |
|---|---|
| **Input** | Approved `assembled.blend`, `usd.root_prim_path` from `configs/base.yaml` |
| **Output** | `robot.usda`, `robot_prim_map.json`, `textures/` |

**Logic:** Exports the Blender scene to USD with Z-up orientation for Isaac Sim. Embedded textures are extracted to `textures/`.

```bash
blender --background --python tool_scripts/blender_export_usd.py -- \
  --blend iterations/006/assembled.blend \
  --output robot.usda \
  --root-prim-path /World/Robot
```

---

## blender_measure_glbs.py

| | |
|---|---|
| **Input** | `component_glbs/` |
| **Output** | `component_dims.json` |

**Logic:** Imports each GLB, computes its world-space bounding box, and writes raw sizes. Called automatically by `generate_components.py`.

---

## common.py

Shared I/O, config, and validation helpers imported by every tool script. Also exposes a JSON-schema CLI used by subagents after writing, and by the orchestrator for `render_views.json`.

```bash
python3 tool_scripts/common.py \
  --schema schemas/assembly.schema.json \
  --data .intermediate/dishwasher/001/iterations/001/assembly.json
```

---

## Blender invocation

Arguments must come after `--`:

```bash
blender --background --python tool_scripts/<script>.py -- --your-args here
```

Override the binary path in `configs/base.yaml` → `paths.blender_binary` if needed.
