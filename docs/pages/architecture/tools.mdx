# Tool Scripts

Not everything in Dexter needs an AI agent. Calling image APIs, converting PNGs to 3D meshes, measuring files, building Blender scenes, and applying critic corrections are fully deterministic. For all of that, Dexter uses plain Python and Blender scripts.

These live in `tool_scripts/`. The orchestrator runs them via bash as needed. API keys always come from environment variables — never from config files.

| Script | What it does | Key input → output |
|--------|-------------|-------------------|
| `generate_components.py` | PNGs, GLBs, and dims for all parts | `parts.json` + `source.png` → `component_images/`, `component_glbs/`, `component_dims.json` |
| `initialize_placement.py` | Pre-computes scales and first layout | `parts.json` + dims → `placement_init.json`, `iterations/001/assembly.json` |
| `apply_critic.py` | Applies critic corrections to prior layout | `assembly.json` + `critic.json` → next `assembly.json` |
| `compute_world_dims.py` | Per-part world size, centre, bbox, parent scale | `assembly.json` + dims → `world_dims.json` |
| `blender_assemble.py` | Builds a Blender scene from the layout | `assembly.json` → `assembled.blend` |
| `blender_render_views.py` | Renders four diagnostic views | `assembled.blend` → `renders/*.png` |
| `blender_export_usd.py` | Exports the final USD asset | `assembled.blend` → `robot.usda` |
| `validate_json.py` | Validates an artifact against its schema | JSON + schema → pass or fail |

---

## generate_components.py

Runs after the parts list is approved. Reads `config.yaml` and `parts.json`, generates one PNG per part via OpenAI, one GLB per part via fal.ai, and measures each GLB with Blender into `component_dims.json`. Skips parts whose outputs already exist.

```bash
python3 tool_scripts/generate_components.py --run-dir .intermediate/dishwasher/001
```

Requires `OPENAI_API_KEY` and `FAL_KEY`.

---

## initialize_placement.py

Combines geometry hints from `parts.json` with measured mesh sizes from `component_dims.json` to produce scale factors and poses for every part. Writes `placement_init.json` and the first `iterations/001/assembly.json`.

```bash
python3 tool_scripts/initialize_placement.py --run-dir .intermediate/dishwasher/001
```

---

## apply_critic.py

From iteration 002 onward, applies critic corrections deterministically: adds position deltas, multiplies scale, adds rotation deltas. Skips `locked` components.

```bash
python3 tool_scripts/apply_critic.py \
  --prev-assembly iterations/001/assembly.json \
  --critic        iterations/001/critic.json \
  --output        iterations/002/assembly.json
```

---

## compute_world_dims.py

Computes per-part world dimensions from `assembly.json` and raw GLB dims. Output feeds the critic so it can judge scale and position without recomputing transforms.

```bash
python3 tool_scripts/compute_world_dims.py \
  --assembly iterations/001/assembly.json \
  --dims     component_dims.json \
  --output   iterations/001/world_dims.json
```

---

## blender_assemble.py

Imports every GLB in `assembly.json`, applies transforms, and builds the parent-child hierarchy as `assembled.blend`.

```bash
blender --background --python tool_scripts/blender_assemble.py -- \
  --layout iterations/001/assembly.json \
  --output iterations/001/assembled.blend
```

---

## blender_render_views.py

Reads `render_views.json` (written by the orchestrator) and renders four cameras: front, top, left, and isometric.

```bash
blender --background --python tool_scripts/blender_render_views.py -- \
  --blend iterations/001/assembled.blend \
  --cameras iterations/001/render_views.json \
  --output-dir iterations/001/renders/
```

---

## blender_export_usd.py

Exports the approved `assembled.blend` to `robot.usda` with textures in `textures/` and a `robot_prim_map.json` mapping Blender object names to USD prim paths.

```bash
blender --background --python tool_scripts/blender_export_usd.py -- \
  --blend iterations/005/assembled.blend \
  --output robot.usda \
  --root-prim-path /World/Robot
```

---

## validate_json.py

Checks subagent and orchestrator JSON output against the corresponding schema.

```bash
python3 tool_scripts/validate_json.py \
  --schema schemas/assembly.schema.json \
  --data .intermediate/dishwasher/001/iterations/001/assembly.json
```

---

## A note on Blender scripts

All Blender scripts follow the same invocation pattern. Arguments must come after `--`:

```bash
blender --background --python tool_scripts/<script>.py -- --your-args here
```

The `blender` binary path defaults to whatever is on your `PATH`, but you can override it in `config.yaml` under `paths.blender_binary`.
