# Schemas

Every pipeline artifact is validated against a JSON Schema in `schemas/`. Validation prevents bad data from propagating downstream.

## Validation command

```bash
python3 tool_scripts/common.py \
  --schema schemas/<name>.schema.json \
  --data <path/to/artifact.json>
```

- Prints `OK` and exits 0 when valid
- Prints each schema error and exits 1 when invalid

## Schema index

| Schema | Artifact | Written by | When validated |
|--------|----------|-----------|----------------|
| `parts.schema.json` | `parts.json` | analyze subagent | After analyze step |
| `assembly.schema.json` | `assembly.json` | initialize_placement.py / update_placement.py | Each iteration |
| `critic.schema.json` | `critic.json` | critic subagent | Each critique |
| `placement_init.schema.json` | `placement_init.json` | initialize_placement.py | After placement init |
| `render_views.schema.json` | `render_views.json` | orchestrator agent | Before rendering |
| `component_dims.schema.json` | `component_dims.json` | blender_measure_glbs.py | After component generation |

## Retry policy

When validation fails, the orchestrator re-invokes the same subagent with errors appended. Repeats up to `loop.max_validation_retries` (default: 2).

## parts.schema.json

Top-level: `object` (string), `parts` (array, min 1).

Per part — always required: `name`, `description`, `parent`, `joint_type`.

Root part requires: `world_size` `[w,d,h]` in metres.

Non-root parts require: `world_size` `[w,d,h]`, `world_center` `[x,y,z]`, `euler_deg` `[rx,ry,rz]`.

## assembly.schema.json

Top-level: `root`, `robot_name`, `parts` (array, min 1).

Per part — all required: `name`, `parent`, `visual_mesh`, `collision_mesh`, `world_size`, `world_center`, `euler_deg`.

- `world_size` — `[W, D, H]` in metres (all axes > 0)
- `world_center` — `[x, y, z]` in metres, world-space centre at the current pose
- `euler_deg` — `[rx, ry, rz]` degrees, XYZ Euler. Negative Z opens a left-hinged door; positive Z opens a right-hinged door; negative X drops a bottom-hinged door forward

## critic.schema.json

Required: `iteration` (int ≥1), `score` (0–100), `pass` (bool), `summary`, `issues`, `per_view_notes`.

Per issue — required: `part`, `problem`. Optional corrections:

- `locked` (bool) — skip this part in `update_placement.py`
- `corrected_world_size` `[W, D, H]` — target world dimensions in metres
- `corrected_world_center` `[x, y, z]` — target world-space centre in metres
- `suggested_rotation_delta` `[drx, dry, drz]` — degrees to add to `euler_deg`

## placement_init.schema.json

Required: `object`, `root_name`, `root_world_size`, `root_world_center`, `root_node_scale`, `parts`.

Per part hint: `name`, `joint_type`, `node_scale`, `pose` (`world_center`, `node_origin`, `euler_deg`). Optional: `children` (recursive).

## render_views.schema.json

Required: `resolution` `[w,h]`, `samples`, `engine`, `file_format`, `cameras`.

Per camera — required: `name`, `direction` `[x,y,z]`, `output`, `light_energy`, `light_type`.

**Do not add `location` or `look_at`** — the render script auto-frames.

## component_dims.schema.json

Required: `parts` (object map keyed by part name).

Per part: `size`, `center`, `min`, `max` — each `[x,y,z]` arrays in raw GLB coordinate space.

## Validate a full run

```bash
RUN=.intermediate/dishwasher/001

python3 tool_scripts/common.py --schema schemas/parts.schema.json --data $RUN/parts.json
python3 tool_scripts/common.py --schema schemas/placement_init.schema.json --data $RUN/placement_init.json

for iter in $RUN/iterations/*/; do
  python3 tool_scripts/common.py --schema schemas/assembly.schema.json --data $iter/assembly.json
  python3 tool_scripts/common.py --schema schemas/critic.schema.json --data $iter/critic.json
done
```
