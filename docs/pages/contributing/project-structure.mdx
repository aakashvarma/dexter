## Key files explained

### opencode.json

Defines all agents: mode (primary/subagent), temperature, prompt file path, and permissions (edit, bash, read, task, webfetch).

The orchestrator agent has broad bash permissions (`python*`, `blender*`). Subagents are tightly scoped.

### config.yaml

Read by the orchestrator agent at run start. Contains:

- `paths` вЂ” directories and blender binary
- `loop` вЂ” iteration exit conditions
- `image_generation` вЂ” OpenAI model settings
- `fal` вЂ” fal.ai 3D model settings
- `placement_init` вЂ” paths for `initialize_placement.py`
- `render` вЂ” Blender render defaults
- `usd` вЂ” USD export settings

### AGENTS.md

Concise pipeline semantics for agents: step order, gates, gotchas. When in doubt, trust `opencode.json` and agent prompts over `AGENTS.md`.

### .opencode/agents/*.md

The actual instructions each agent follows. The orchestrator agent prompt is the most detailed вЂ” it specifies exact step ordering and gate behavior.

### schemas/

JSON Schema Draft 2020-12 files. All use `additionalProperties: false`.

### tool_scripts/

Python scripts (stdlib + requirements.txt deps) and Blender scripts (`import bpy`). Blender scripts parse args after `--` in `sys.argv`.

## What is NOT in the repo

| Missing | Why |
|---------|-----|
| Test suite | No CI; validate with `validate_json.py` + manual runs |
| Python package manifest | Only `requirements.txt` |
| Pipeline driver script | OpenCode orchestrator agent is the only entrypoint |
| API keys | Environment variables only |
| Run artifacts | `.intermediate/` is gitignored |
