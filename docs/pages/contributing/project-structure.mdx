## Key files explained

### opencode.json

Defines all agents: mode (primary/subagent), temperature, prompt file path, and permissions (edit, bash, read, task, webfetch).

The orchestrator has broad bash permissions (`python*`, `blender*`). Subagents are tightly scoped.

### config.yaml

Read by the orchestrator at run start. Contains:
- `paths` — directories and blender binary
- `loop` — iteration exit conditions
- `image_generation` — OpenAI model settings
- `fal` — fal.ai 3D model settings
- `render` — Blender render defaults
- `usd` — USD export settings

Models are **not** configured here — subagents use the logged-in OpenCode model.

### AGENTS.md

Written by `opencode /init`. Concise pipeline semantics for agents: step order, gates, gotchas. When in doubt, trust `opencode.json` and agent prompts over `AGENTS.md`.

### .opencode/agents/*.md

The actual instructions each agent follows. The orchestrator prompt is the most detailed — it specifies exact config JSON formats, step ordering, and gate behavior.

### schemas/

JSON Schema Draft 2020-12 files. All use `additionalProperties: false`. Field `description` strings document pipeline semantics and sometimes embed transform formulas.

### tool_scripts/

Python scripts (stdlib + requirements.txt deps) and Blender scripts (`import bpy`). Blender scripts parse args after `--` in `sys.argv`.

Docstrings at the top of each script document: what it does, run command, and JSON config schema.

## Run directory (.intermediate/)

Gitignored. Structure per run:

```
.intermediate/<asset>/<NNN>/
  source.png
  parts.json                     # Parts IR
  prompts.json
  component_dims.json
  placement_hints.json
  placement.confirmed            # human-approved iteration id
  build_component_prompts.json   # orchestrator-written configs
  openai_imagegen.json
  fal_image_to_3d.json
  component_images/
  component_glbs/
  iterations/<NNN>/
    assembly.json                # Layout IR
    assembled.blend
    render_views.json
    renders/
    critic.json                  # Critique IR
  robot.usda
  robot_prim_map.json
  textures/
```

## What is NOT in the repo

| Missing | Why |
|---------|-----|
| Test suite | No CI; validate with `validate_json.py` + manual runs |
| Python package manifest | Only `requirements.txt` |
| Pipeline driver script | OpenCode orchestrator is the only entrypoint |
| API keys | Environment variables only |
| Run artifacts | `.intermediate/` is gitignored |
