# Dishwasher Run

This page traces a complete Dexter v0 run on a product photo of a dishwasher — from part identification through placement convergence to final export. The run produced four moving parts: **cabinet_body**, **front_door**, **upper_rack**, and **lower_rack**.

## Run command

```bash
opencode run --agent orchestrator -- "build the dishwasher from input_images/dishwasher.png"
```

Output directory: `.intermediate/dishwasher/001/`

---

## Step 1 — Part identification

The **analyze** subagent read the reference image and proposed four moving parts with joint types and geometry hints. After human approval of the Parts IR, the run continued.

![Reference product photo of a dishwasher](/assets/images/dexter/dishwasher_input.png)

*Reference input image used for this run.*

**Parts identified:**

| Part | Joint type | Parent | Key geometry |
|------|-----------|--------|-------------|
| `cabinet_body` | fixed | — (root) | Full appliance body |
| `front_door` | revolute | cabinet_body | `hinge_side: bottom`, full-width door |
| `upper_rack` | prismatic | cabinet_body | `slide_axis: -y`, upper rack |
| `lower_rack` | prismatic | cabinet_body | `slide_axis: -y`, lower rack |

The human parts gate paused here for review before any 3D generation began.

## Step 3 — Mesh generation

The orchestrator ran image-to-3D on each render via fal.ai Hunyuan 3D, producing one GLB per part: `cabinet_body.glb`, `front_door.glb`, `upper_rack.glb`, `lower_rack.glb`.

Blender then measured raw bounding boxes → `component_dims.json`. `compute_placement_scales.py` pre-computed placement hints with `--root-world-dims 0.60 0.60 0.85`.

## Steps 4–5 — Placement and critique loop

Each round, the **placement** subagent wrote positions, orientations, and scales into the Layout IR. Blender assembled the parts and rendered four diagnostic views. The **critic** subagent scored the result against the reference image and returned structured fixes.

The score moved from **45** at iteration 1 to **82** at iteration 6 (internal 0–100 scale).

### Iteration 1 — significant errors

At iteration 1 the assembly had major problems:
- Door was **upright** instead of open and flat
- Both racks **extended past the body edges**
- Lower rack was **not resting on the door**
- Multiple **mesh intersections**

![Placement iteration 1 isometric view](/assets/images/dexter/iter_001_isometric.png)

*Iteration 1: door upright, racks extend past body, parts intersect.*

![Placement iteration 1 collision visualization](/assets/images/dexter/iter_001_collisions.png)

*Iteration 1 collision view in Blender. Highlighted areas show mesh intersections flagged by the critic.*

The critic flagged rotation, scale, and collision issues. The placement agent applied corrections on the next round.

### Iteration 6 — converged layout

By iteration 6 the layout matched the reference photo:
- Door lay **flat and open** below the racks
- Both racks **seated inside the body** without intersecting walls
- Proportions **consistent** with the source image
- Critic **locked** the main links; only minor depth adjustments flagged

### Score progression

| Iteration | Score | Main issues |
|-----------|-------|-------------|
| 1 | 45 | Door upright, rack overflow, collisions |
| 2 | 58 | Door rotation improving, rack scale wrong |
| 3 | 68 | Door flat, upper rack position off |
| 4 | 74 | Rack pull-out distance too short |
| 5 | 79 | Minor depth adjustments |
| 6 | 82 | Main links locked, minor depth only |

Loop stopped: score 82 with `no_improvement_patience` after iteration 5–6 plateau.

## Step 6 — Export

After human placement gate approval (`placement.confirmed` → `006`), the orchestrator exported USD:

```bash
blender --background --python tool_scripts/blender_export_usd.py -- \
  --blend .intermediate/dishwasher/001/iterations/006/assembled.blend \
  --output .intermediate/dishwasher/001/robot.usda \
  --root-prim-path /World/Robot
```

## Lessons from this run

1. **Parts gate matters** — approving the four-part decomposition before 3D generation saved expensive API calls on wrong parts.
2. **Placement hints are essential** — `compute_placement_scales.py` gave iteration 1 a reasonable starting point (score 45 vs likely below 20 without hints).
3. **Critic drives convergence** — structured per-component fixes (`suggested_rotation_delta`, `suggested_scale_factor`) let placement improve systematically.
4. **Human placement gate** — reviewing iteration 6 renders before USD export caught a depth issue the critic scored as minor.
5. **Six iterations is typical** — for a 4-part object with open door and extended racks, 5–6 iterations to reach score ~80 is normal with default `config.yaml` settings.
